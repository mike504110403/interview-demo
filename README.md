# 專案建置經驗分享

專業的Web簡報系統，展示從零到一的專案建置經驗。

## 簡報結構

### 單一HTML，雙簡報系統（26頁）

**透過上方Tab切換：**

#### 主簡報（18頁）
1. 封面
2-7. BC專案（背景、核心功能、技術棧、架構、挑戰、優化）
8-10. 專案成果三部曲（理想→現實→MVP）
11-12. AI方法論與技術決策
13-16. 健身App實戰案例（含App Store QR Code）
17. 應用到直播平台
18. 感謝聆聽

#### 直播技術簡報（8頁）
19. 封面
20. 直播技術對比（RTMP vs HLS vs WebRTC + 成本分析）
21. 混合方案 - 降低成本的關鍵（主播連麥用SFU + 觀眾用CDN）
22. WebRTC 架構對比表格（P2P vs MCU vs SFU）
23. 技術選型：自建 vs SaaS（含 Agora/聲網說明）
24. Demo專案展示（WebRTC + Stream Demo）
25. 外包專案接手策略（評估優於重做）
26. 感謝聆聽

### 視覺風格
- 深色專業配色（深藍 + 深灰）

## 快速開始

### 方式一：直接開啟
```bash
open index.html
```

### 方式二：使用腳本（推薦）
```bash
./start.sh
```

### 方式三：手動啟動伺服器
```bash
python3 -m http.server 8000

# 然後在瀏覽器開啟
# http://localhost:8000/
```

## 使用說明

### Tab切換
- 點擊頂部「主簡報」按鈕：跳到第1頁，開始主簡報
- 點擊頂部「直播技術」按鈕：跳到第19頁，開始直播技術簡報
- 在任何頁面都可以隨時切換

### 導航方式
- **Tab切換**：頂部Tab按鈕
- **上/下頁**：右側導航按鈕或鍵盤方向鍵
- **頁碼顯示**：僅顯示當前簡報的頁數（例：主簡報顯示 1/18，直播技術顯示 1/7）

## 鍵盤快捷鍵

- `→` / `↓` / `空格` - 下一頁（當前簡報）
- `←` / `↑` - 上一頁（當前簡報）
- `Home` - 第一頁
- `End` - 最後一頁
- `F11` - 全螢幕切換
- `Esc` - 退出全螢幕

## 開發說明

### 修改內容
直接編輯HTML檔案即可修改簡報內容

### 調整樣式
編輯 `css/style.css`，使用CSS變數統一管理配色

### 添加功能
在 `js/main.js` 中添加新功能

### 添加新簡報
1. 在HTML中添加新的`<section>`，設定`data-presentation="your-name"`
2. 在Tab導航添加新按鈕
3. JavaScript會自動處理切換邏輯
